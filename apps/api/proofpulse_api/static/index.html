AWS Synergy

Why AWS Activate Credits Matter

Running a production-ready, globally distributed decentralized oracle and RPC monitoring grid requires resilient cloud architecture:

  • Continuous probes on EC2/Lambda: High-frequency monitoring nodes deployed globally to detect millisecond latency changes in Web3 endpoints.
  • S3 evidence storage: Highly durable, tamper-resistant bucket configuration to host canonical JSON evidence files for on-chain reference.
  • CloudWatch/OpenSearch ingestion: Aggregating massive telemetry feeds from multiple targets to construct real-time performance profiles.
  • Bedrock AI SRE analysis: Running secure, hosted Llama3/Claude models to analyze logs, deduce root causes, and output non-obvious recommendations.
  • Multi-region Web3 infrastructure monitoring: Ensuring high-availability probes across EU, US, and AP regions to eliminate local network false positives.
  • Testnet/Mainnet relayer costs: Subsidizing gas and server costs to submit attestations onto Sepolia/Base testnets and mainnet environments.
Step-by-step Flow

Submission-Ready Demo Flow

ProofPulse operates systematically to turn infrastructure alerts into verifiable cryptographic evidence:

1

Monitor Web3 Target

ProofPulse nodes actively ping RPC, oracle, bridge, or validator endpoints from distributed AWS locations.

2

Detect Degraded RPC/Oracle Behavior

If latency exceeds threshold (e.g. >420ms) or error rate climbs, check statuses mark the target as degraded.

3

AI SRE Creates Incident Analysis

Bedrock evaluates logs and metrics, computes a unified risk score, compiles a summary, and lists recovery actions.

4

Evidence Bundle Hash is Computed

A deterministic canonical JSON representation of targets, incidents, and raw checks is compiled, computing a unique bundle hash.

5

On-chain Attestation is Prepared

A payload containing the cryptographic hash and metadata is packaged for Sepolia EVM broadcast.
