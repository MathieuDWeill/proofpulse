# Winning strategy

## Positioning

Do not pitch ProofPulse as a generic monitoring dashboard. Pitch it as **verifiable reliability infrastructure for institutional Web3**.

The strongest sentence is:

> Web3 infrastructure teams already monitor uptime, but they cannot prove the evidence trail behind an incident. ProofPulse turns cloud telemetry and AI incident analysis into tamper-evident on-chain proofs.

## Why judges should care

- It is AWS-native by design, so the credit ask is credible.
- It has a clear enterprise/Web3 buyer: infra teams, validators, RWA platforms, bridge teams, custody providers.
- It uses AI in a non-gimmicky way: incident summarization and risk analysis tied to evidence.
- It uses blockchain in a non-gimmicky way: public proof hash, private evidence.
- It can be demoed in two minutes.

## What to emphasize

1. **Trust gap:** status pages and dashboards are not audit trails.
2. **Evidence:** the system creates canonical evidence bundles.
3. **Privacy:** only hashes go on-chain.
4. **AWS:** the product actually consumes infra credits meaningfully.
5. **Expansion:** RWA, bridges, validators, RPC providers, custody, DeFi.

## What not to over-explain

- Do not spend too much time on contract internals.
- Do not claim production-grade AI compliance yet.
- Do not say it replaces Datadog. Say it complements monitoring with verifiable evidence.

## Final submission checklist

- GitHub repo public.
- README screenshot or dashboard image.
- Demo video link.
- API deployed or local demo recorded.
- Contract address if possible; otherwise prepared calldata shown.
